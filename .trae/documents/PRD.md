# 全球供应链风险监控系统 - 管理后台 PRD

## 1. 产品概述

面向企业运维团队和管理层的供应链风险监控后台。系统通过角色权限控制，确保不同层级人员看到与其职责匹配的功能入口和数据。

## 2. 角色定义

### 2.1 普通运维（role: operator）
- **职责**：处理本团队分配的工单，执行预案，查看报表
- **可见菜单**：
  - 📋 团队工单（默认首页）
  - 📝 预案管理
  - 📊 报表统计

### 2.2 主管（role: supervisor）
- **职责**：查看所有工单、审计日志、管理系统用户与配置
- **可见菜单**（全量）：
  - 📊 监控大盘（默认首页）
  - 📋 全量工单
  - ⚠️ 异常事件
  - 📝 预案管理
  - 📊 报表统计
  - 🔍 日志查询
  - 📜 审计日志
  - ⚙️ 系统管理
    - 👥 用户管理
    - 🔐 角色权限
    - 📁 系统配置

## 3. 核心功能需求

### 3.1 登录与身份识别
- 支持用户名密码登录
- 登录成功后返回用户信息（含 role 字段）
- 登录状态持久化（localStorage）
- 支持退出登录

### 3.2 侧边栏菜单动态渲染
- **严格按角色渲染**：不同角色看到不同菜单项
- **切换账号/角色后立即刷新**：清除旧菜单，使用新角色重新生成
- **无权限页面自动跳转**：当前页面新角色不能看时，跳到新角色默认首页

### 3.3 路由守卫（地址栏拦截）
- 仅检查登录态 → **检查登录态 + 角色权限**
- 普通运维访问 `/audit`、`/system/user`、`/log`、`/exception` → 拦截并跳回 `/workorder`（团队工单）
- 主管可正常进入所有页面
- 未登录 → 跳 `/login`

### 3.4 登录后默认落点
| 角色 | 默认首页路径 |
|------|------------|
| operator（运维） | `/workorder`（团队工单） |
| supervisor（主管） | `/dashboard`（监控大盘） |
- 刷新页面（F5）仍按上述规则判断

## 4. 页面清单

| 路径 | 页面 | 允许角色 |
|------|------|---------|
| `/login` | 登录页 | 所有人（未登录） |
| `/dashboard` | 监控大盘 | supervisor |
| `/workorder` | 团队工单 | operator, supervisor |
| `/workorder/all` | 全量工单 | supervisor |
| `/exception` | 异常事件 | supervisor |

## 5. 交互细节

### 5.1 切换账号/角色流程
1. 用户点击「退出登录」或切换账号
2. 清除本地用户信息 + 权限菜单缓存
3. 回到登录页
4. 新账号登录 → 立即用新角色渲染菜单
5. **关键**：绝不沿用上次登录的菜单配置

### 5.2 刷新页面流程
1. 页面加载 → 读取 localStorage 用户信息
2. 有登录态 → 获取 role → 检查当前路由是否有权限
3. 有权限 → 正常渲染 + 按 role 生成菜单
4. 无权限 → 跳转到该角色默认首页
5. 无登录态 → 跳登录页

### 5.3 地址栏直链拦截
1. 路由守卫（beforeEach）触发
2. 取 to.path → 查路由表的 meta.roles
3. 当前 user.role 在 meta.roles 中 → 放行
4. 不在 → `next(该角色的默认首页路径)`

## 6. 非功能需求
- **响应速度**：菜单切换 < 100ms，路由跳转 < 200ms
- **安全性**：仅前端拦截为第一道防线，真实接口需配合后端鉴权
- **兼容性**：Chrome / Edge / Firefox 最新两个版本
